# /swarm Command

Manage Claude Flow v3 multi-agent swarm orchestration.

## Usage

When the user runs `/swarm`, perform the following based on the subcommand:

### `/swarm init`
Initialize a new swarm for the current task:
1. Determine the optimal topology based on the task (hierarchical for complex, mesh for parallel)
2. Run `npx ruflo swarm init --topology <type> --max-agents <n>`
3. Spawn the required agents based on the task type

### `/swarm status`
Show current swarm status:
1. Run `npx ruflo status`
2. Run `npx ruflo swarm status`
3. Present a summary of active agents, pending tasks, and memory state

### `/swarm spawn <type>`
Spawn a new agent in the current swarm:
1. Run `npx ruflo agent spawn -t <type> --name <auto-generated-name>`
2. Confirm the agent is active

### `/swarm doctor`
Run diagnostics on the swarm setup:
1. Run `npx ruflo doctor`
2. Report any issues found

### `/swarm end`
Gracefully shut down the swarm:
1. Run `npx ruflo swarm stop`
2. Persist session metrics

## Default (no subcommand)
Show a brief status overview and suggest next actions.
